Here is your **Laptop A (HQ Receiver) Only Playbook**. This machine acts as the central cloud hub that receives data from all store POS laptops and stores it in a unified database.

---

### Step 1: Prep Laptop A (One-Time Setup)

1. Open **Command Prompt** (as Administrator, if possible).
2. Install Flask (the web framework that will receive the data):
   ```cmd
   pip install flask
   ```
3. **Find your IP Address** (Laptop B needs this to find you):
   - In the same Command Prompt, type:
     ```cmd
     ipconfig
     ```
   - Look for your active network adapter (usually **Wi-Fi** or **Ethernet**).
   - Find the **IPv4 Address** (e.g., `192.168.1.100` or `10.0.0.5`). 
   - **Write this down exactly.** You will give this to Laptop B.

---

### Step 2: Create the HQ Receiver Script (`hq_receiver.py`)

Create a new folder on your desktop (e.g., `HQ_Retail_OS`) and inside it, create a new file called **`hq_receiver.py`**.

**Copy and paste this exact code:**

```python
import sqlite3
import json
import datetime
from flask import Flask, request, jsonify

app = Flask(__name__)

# --- 1. Setup Central Database (Runs automatically on startup) ---
def init_database():
    conn = sqlite3.connect('central_hq.db')
    cursor = conn.cursor()

    # Stores the header of each sale
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS central_sales (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            store_id TEXT,
            sale_id INTEGER,
            timestamp TEXT,
            total_amount REAL,
            received_at TEXT,
            UNIQUE(store_id, sale_id)  -- Prevents duplicate uploads
        )
    """)

    # Stores the line items of each sale (we will extend this later)
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS central_sale_items (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            store_id TEXT,
            sale_id INTEGER,
            product_id INTEGER,
            quantity INTEGER,
            subtotal REAL,
            received_at TEXT
        )
    """)

    # Stores current inventory snapshots from each store
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS central_products (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            store_id TEXT,
            product_id INTEGER,
            barcode TEXT,
            name TEXT,
            price REAL,
            stock INTEGER,
            updated_at TEXT,
            UNIQUE(store_id, product_id)
        )
    """)

    conn.commit()
    conn.close()
    print("✅ Central HQ Database initialized (central_hq.db)")

# --- 2. The Ingestion Endpoint (Where Laptop B sends data) ---
@app.route('/ingest', methods=['POST'])
def ingest_sale():
    try:
        data = request.get_json()
        store_id = data.get('store_id')
        sale_id = data.get('sale_id')
        timestamp = data.get('timestamp')
        total_amount = data.get('total_amount')

        # Basic validation
        if not all([store_id, sale_id, timestamp, total_amount is not None]):
            return jsonify({"error": "Missing required fields"}), 400

        received_at = datetime.datetime.now().isoformat()

        # Connect to the central database
        conn = sqlite3.connect('central_hq.db')
        cursor = conn.cursor()

        # Insert the sale (ignore if already exists to handle retries)
        cursor.execute("""
            INSERT OR IGNORE INTO central_sales 
            (store_id, sale_id, timestamp, total_amount, received_at)
            VALUES (?, ?, ?, ?, ?)
        """, (store_id, sale_id, timestamp, total_amount, received_at))

        # Check if this was a new insertion or a duplicate
        if cursor.rowcount == 0:
            conn.close()
            return jsonify({"message": "Duplicate sale ignored"}), 200

        conn.commit()
        conn.close()

        print(f"📥 Received Sale #{sale_id} from {store_id} | Total: ${total_amount}")
        return jsonify({"status": "success", "sale_id": sale_id}), 200

    except Exception as e:
        print(f"❌ Error processing request: {e}")
        return jsonify({"error": str(e)}), 500

# --- 3. Optional: Inventory Sync Endpoint (for future expansion) ---
@app.route('/ingest/inventory', methods=['POST'])
def ingest_inventory():
    """
    Future endpoint: Laptop B can send its full product list here.
    Not required for Phase 1, but good to have the skeleton.
    """
    return jsonify({"message": "Inventory sync not yet implemented"}), 501

# --- 4. Health Check (So Laptop B knows you are alive) ---
@app.route('/health', methods=['GET'])
def health_check():
    return jsonify({"status": "HQ is alive"}), 200

# --- 5. Start the Server ---
if __name__ == '__main__':
    init_database()
    print("🚀 HQ Receiver is now running...")
    print(f"📡 Listening on all network interfaces (0.0.0.0:5000)")
    print(f"👀 Waiting for data from Store POS laptops...")
    # host='0.0.0.0' makes it accessible from other laptops on the same network
    app.run(host='0.0.0.0', port=5000, debug=True, use_reloader=False)
```

### Step 3: Run the HQ Receiver

1. In your Command Prompt, navigate to your new HQ folder:
   ```cmd
   cd "C:\Users\YourName\Desktop\HQ_Retail_OS"
   ```
2. Run the script:
   ```cmd
   python hq_receiver.py
   ```
3. **You should see this output:**
   ```
   ✅ Central HQ Database initialized (central_hq.db)
   🚀 HQ Receiver is now running...
   📡 Listening on all network interfaces (0.0.0.0:5000)
   👀 Waiting for data from Store POS laptops...
   * Running on http://0.0.0.0:5000
   ```

**⚠️ Critical:** Keep this Command Prompt window **open and running**. If you close it, Laptop B cannot send data. 
(Do not press `Ctrl+C` unless you want to stop the server.)

### Step 4: Tell Laptop B Your IP Address

Give the **IPv4 address** you found in Step 1 to the person setting up Laptop B. 
Tell them to put it into their `store_sync_agent.py` file at Line 9. For example, if your IP is `192.168.1.100`, they must set:
```python
HQ_API_URL = "http://192.168.1.100:5000/ingest"
```

### Step 5: Verify Data is Arriving

Once Laptop B sends its first sale, you will see a log appear in your running Command Prompt:
```
📥 Received Sale #1 from STORE_001 | Total: $4.99
```

To double-check the data is safely stored in your central database, open a **new Command Prompt** window (leave the Flask one running) and navigate to your HQ folder. Then run:

```cmd
python -c "import sqlite3; conn=sqlite3.connect('central_hq.db'); cursor=conn.cursor(); cursor.execute('SELECT store_id, sale_id, total_amount, timestamp FROM central_sales'); print(cursor.fetchall()); conn.close()"
```

You should see a list of tuples containing the sales data from Laptop B.

### Step 6: Keep it Running Permanently (Optional - Auto-Start on Boot)

If you want the HQ receiver to start automatically whenever Laptop A restarts (without you having to manually open the terminal):

1. Press **`Win + R`**, type `shell:startup`, and press Enter. This opens the Startup folder.
2. Right-click inside the folder, select **New → Shortcut**.
3. **Location of the item:** Type:
   ```
   cmd /k "cd /d C:\Users\YourName\Desktop\HQ_Retail_OS && python hq_receiver.py"
   ```
   *(Make sure you replace the folder path with your actual path!)*
4. Name it `HQ_Receiver` and click Finish.

Now, every time Laptop A turns on, this command prompt will automatically pop up and start listening for data.
