Here is your **AROS Desktop App Playbook**. This is the local Electron app that
turns your real `pos-system/pos_system.db` sales data into an Insights
dashboard (top/bottom performers, trend shifts). Everything runs on this one
machine — nothing is sent anywhere.

---

### Step 1: One-Time Setup

1. Make sure you have **Python 3.10+** and **Node.js 18+** installed.
   ```cmd
   python --version
   node --version
   ```
2. Install the backend's Python dependencies:
   ```cmd
   cd "C:\All projects\Hackathon dindigul\Open-Source-Point-of-Sale-System\stitch_electron_os_interface\aros_backend"
   pip install -r requirements.txt
   ```
3. Make sure `pos-system/pos_system.db` actually has data in it. If it's a
   fresh/empty database, seed it first:
   ```cmd
   cd "C:\All projects\Hackathon dindigul\Open-Source-Point-of-Sale-System\pos-system"
   python -c "from database import POSDatabase; POSDatabase()"
   python seed_data.py
   ```
4. Install the Electron app's dependencies:
   ```cmd
   cd "C:\All projects\Hackathon dindigul\Open-Source-Point-of-Sale-System\stitch_electron_os_interface\electron"
   npm install
   ```
   This downloads Electron itself — it can take a few minutes the first time.

---

### Step 2: Run the App

```cmd
cd "C:\All projects\Hackathon dindigul\Open-Source-Point-of-Sale-System\stitch_electron_os_interface\electron"
npm start
```

This does two things automatically:
- Spawns the Python backend (`aros_backend/app.py`) on `127.0.0.1:8000`.
- Opens an Electron window pointed at it.

**Keep the window open** — closing it also shuts down the backend process
running behind it.

---

### Step 3: First-Run Walkthrough

1. **Create account** — the very first launch shows a "Create a local
   account" screen. Pick any username/password; it's stored only in
   `aros_backend/aros_app.db` on this machine (PBKDF2-hashed).
2. **Define your categories** — you'll see your real 25 products already
   grouped into 3 suggested categories (Produce & Bakery, Dairy/Meat & Deli,
   Pantry & Household). You can:
   - Rename a group by editing its title field.
   - Move any product into a different group (or a brand-new one) using the
     dropdown next to it.
   Click **CONFIRM CATEGORIES** when done.
3. **Insights dashboard** — you land here automatically. It shows real
   numbers computed from `pos_system.db`: total revenue, stockouts, a
   Pareto (80/20) breakdown of which categories/products drive revenue, and
   week-over-week trend shifts. Use the filter box to search products, and
   the switch in the sidebar to toggle light/dark mode.
4. **News tab** — real, live retail-industry headlines (free Google News
   RSS, no API key or setup needed). It has its own location dropdown, same
   set of locations as Insights — switching it changes the search query
   (e.g. "Chennai retail business news"). Click a headline to open it in
   your regular web browser.
5. Next time you run `npm start`, you'll go straight to the login screen —
   your account and categories are remembered.

### Step 4 (optional): Multi-location insights (Tamil Nadu shop network)

A dropdown next to REFRESH lets you switch Insights between your single
`pos_system.db` store (USD) and a simulated Tamil Nadu chain (₹) spread
across Dindigul, Madurai, Coimbatore, Tiruchirappalli, and Chennai — each
with its own genuinely different revenue/category mix. The dropdown only
appears once that data exists. To generate it:

```cmd
cd "C:\All projects\Hackathon dindigul\Open-Source-Point-of-Sale-System\pos-system"
python tamil_nadu_local_simulator.py --no-live
```

This seeds 45 days of history per shop into `HQ_Retail_OS/central_hq.db`
(read-only from AROS's side, same as `pos_system.db`) and exits. Drop
`--no-live` to keep it running afterward and stream new sales continuously
(Ctrl+C to stop) — either way, just refresh/reopen Insights in AROS to see
the new locations show up.

### Step 5 (optional): Chat with a local LLM about a report

The **Reports** tab's **SEND TO LLM** button explains a generated report in
plain language and opens a chat window for follow-up questions — using a
small model running entirely on your machine via [Ollama](https://ollama.com),
not any cloud API.

1. Install Ollama from https://ollama.com if you don't have it.
2. Pull the model once:
   ```cmd
   ollama pull phi3
   ```
3. Make sure Ollama is running (the Ollama app running in your system tray
   is enough, or run `ollama serve` in a terminal).
4. Generate a report, then click **SEND TO LLM →** at the bottom of it.

Want a different local model? Set `AROS_OLLAMA_MODEL` before running
`npm start` (e.g. `set AROS_OLLAMA_MODEL=llama3` on Windows), then
`ollama pull` that model too.

### Troubleshooting

| Symptom | Likely cause |
| :--- | :--- |
| `AROS backend unavailable` error dialog on launch | Python dependencies missing — re-run Step 1.2, or check `pip` is on PATH |
| Blank/white window that never loads | The Python backend crashed on start — run `python -m uvicorn app:app` directly from `aros_backend/` to see the real error in the terminal |
| `FileNotFoundError: ...pos_system.db not found` | `pos-system/pos_system.db` doesn't exist yet — run Step 1.3 |
| Wizard shows 0 products | `pos_system.db` exists but is empty — run `python seed_data.py` from `pos-system/` |
| Forgot your local password | Delete `aros_backend/aros_app.db` and restart the app — you'll be prompted to create a new account (this does **not** touch your real sales data) |
| "Couldn't reach Ollama" on SEND TO LLM | Ollama isn't running — start the Ollama app or run `ollama serve` |
| "Model isn't pulled yet" on SEND TO LLM | Run `ollama pull phi3` (or whatever `AROS_OLLAMA_MODEL` you set) |
